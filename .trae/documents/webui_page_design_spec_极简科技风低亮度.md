# WebUI 页面设计规范（极简科技风｜低亮度护眼｜可选深色主题）

## 1) Layout（桌面优先）
- 布局策略：CSS Grid 为主（12 列栅格），Flexbox 作为局部辅助。
- 桌面端（≥1200px）：三分区并排固定结构。
  - 左：图片预览区（4/12）
  - 中：结果区（4/12）
  - 右：参数区（4/12）
- 平板端（768–1199px）：2 列布局（预览+结果并排，参数下沉为整行可折叠）。
- 移动端（<768px）：单列纵向堆叠（预览 → 结果 → 参数），参数默认折叠以降低信息噪声。
- 间距系统：8px 基准（8/16/24/32）。分区之间固定 16px 间距，避免视觉拥挤。

## 3) Global Styles（设计令牌 / 主题）
### 3.1 主题模式
- 默认主题：低亮度科技风（深蓝/深灰、低饱和、低眩光）。
- 可选主题：深色主题（更深背景、更高层级对比，但保持低饱和）。
- 实现建议：用 CSS Variables 定义 token，通过 `data-theme="dim" | "dark"` 切换。

### 3.2 颜色（建议 token）
- 背景 Background
  - --bg: #0B1220（深蓝黑）
  - --bg-elevated: #0F1A2B（面板底）
- 边框/分割线
  - --border: #22324A（低饱和深灰蓝）
- 文字 Text
  - --text: #C7D2E2（偏冷浅灰蓝）
  - --text-muted: #94A3B8（辅助信息）
- 强调 Accent（低饱和蓝）
  - --accent: #3B6FB6
  - --accent-hover: #4A7FC6
- 状态色（尽量克制，避免高亮刺眼）
  - --success: #2F8F6B
  - --warning: #A27B2C
  - --danger: #A24343

### 3.3 字体与排版
- 字体：系统字体优先（Inter / SF Pro / Segoe UI / Noto Sans）。
- 字号层级：
  - H1 20–24 / H2 16–18 / Body 14–16 / Caption 12
- 行高：1.4–1.6（降低阅读疲劳）。

### 3.4 组件风格
- 圆角：8px（面板/卡片/输入框），按钮 10px。
- 阴影：极弱（仅用于层级区分），避免“发光”风格造成眩光。
- Hover/Focus：
  - Hover：背景轻微提亮（+3%~5% 亮度）
  - Focus：1px 低饱和外描边（--accent），禁止高亮荧光效果

## 4) Page Structure（页面结构）
- 顶部栏（Top Bar）：固定高度 56px，内容左右分布。
- 主工作区（Main）：三分区并排，分区内部统一“标题栏 + 内容区”的面板结构。
- 面板结构（Panel）：
  - Panel Header：标题 + 辅助动作（如清空/重置/帮助图标入口）
  - Panel Body：滚动区域（避免整页滚动造成迷失）

## 5) Sections & Components（逐区组件规范）

### 5.1 顶部栏 Top Bar
- 左侧：产品名/页面名（“WebUI 工作台”）。
- 右侧：主题切换（Dim/Dark 切换开关）；可放置少量全局动作（不超过 2 个）。
- 分割：底部 1px --border 分割线。

### 5.2 图片预览区（左）
- Header：区标题“图片预览”。
- Body：
  - 导入控件：支持点击选择文件与拖拽落区（Dropzone）。
  - 预览画布：
    - 默认居中等比适配
    - 支持滚动查看超大图
    - 提供基础查看控件入口（放大/缩小/适配），控件采用低对比图标按钮
- 空状态：展示简洁提示文案（1 行主提示 + 1 行副提示）。

### 5.3 结果区（中）
- Header：区标题“结果”。
- Body：
  - 结果展示：与预览一致的展示规格（等比、居中、可滚动）。
  - 对比查看：提供“并排/切换”二选一控件（默认为切换，减少拥挤）。
  - 导出动作：主次分明（下载为主，其他为次），按钮不使用高亮渐变。
- 空状态：提示“执行后在此显示结果”。

### 5.4 参数区（右）
- Header：区标题“参数”。右侧放“恢复默认”次级按钮。
- Body：
  - 参数表单：输入控件统一高度与对齐；标签左对齐，说明文本使用 --text-muted。
  - 分组/折叠：按逻辑分组（Accordion），默认展开最常用组，其他折叠减少干扰。
  - 校验与提示：错误信息采用 --danger 的低亮度版本；避免大块红色背景。

### 5.5 执行与状态（全局或结果区内）
- 主按钮“执行”：放置在结果区 header 或参数区底部（二选一，保持一致）。
- 运行态：
  - 按钮变为 loading；禁用重复触发
  - 状态提示：细条/小型 badge，不遮挡内容
- 失败态：
  - 在结果区显示简明错误摘要 + “重试”
  - 保留用户已填写的参数，不自动清空

## 6) 交互与可访问性（简要）
- 键盘可达：主题切换、主要按钮、参数输入、导入控件均可 Tab 导航。
- 对比度：在低亮度前提下，正文文字与背景对比度应保持可读（避免过暗）。
- 动效：仅使用 150–200ms 的淡入/高度过渡（折叠、hover），不使用闪烁与强光效。
